OO vs. Functional Programming

Functional Programming

From Wikipedia: "Functional programming emphasizes functions that produce results that depend only on their inputs."

Object Oriented Programming

From Wikipedia: "Object-oriented programming (OOP) is a programming paradigm that represents the concept of "objects" that have data fields (attributes that describe the object) and associated procedures known as methods."

There are your basic definitions. What do they mean? Why use one over the other?

Functional Programming emphasizes functions that produce relults. Object Oriented Programming of course also has functions and is obviously concerned with results, however it emphasizes objects, rather than funcitons. The key difference here is that while both approaches have functions, in OOP the functions are contained within the objects.

When use what?

Food for thought in this from Stack Overflow:

Object-oriented languages are good when you have a fixed set of operations on things, and as your code evolves, you primarily add new things. This can be accomplished by adding new classes which implement existing methods, and the existing classes are left alone. Functional languages are good when you have a fixed set of things, and as your code evolves, you primarily add new operations on existing things. This can be accomplished by adding new functions which compute with existing data types, and the existing functions are left alone.
