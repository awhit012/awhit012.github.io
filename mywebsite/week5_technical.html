Classes

Everything is an object in Ruby

Ruby is what is called an Object Oriented programming language. Everything in Ruby is an "object", which means that it is an entity that the computer knows about, and that it has attributes, or data attached to it.

In this example we have an array, or list of tricks that I can do on a skateboard asigned to the variable 'my_tricks'. Each trick on the list is an object, the list is an object, and the variable that holds the list is an object that holds the list. What about classes?

Classes are what they sound like

In the example above we have created a function difficult? that checks if a trick is difficult or not, based on whether or not it is in my_tricks. We also have a function 'initialize' that creates class variables. All of this is sitting in the class 'Skateboarding.' Classes are what they sound like. They are a way of grouping similar objects together for a purpose. In this case we are grouping two functions and a list all having to do with skateboarding under the 'Skateboarding' class. Since that is everything it doesn't seem to matter much, but what if we complicate this a little more.
